🚀 UMoeAssetManager 的六大核心优势

针对手游热更场景的定制化优化。

① 短名映射系统 — 业务与路径解耦

业务代码只需使用短名，资产移动/重命名只需更新映射文件，无需改动业务代码。

C++ // 业务代码永远不需要知道完整路径 LoadSyncAssetObject<USkeletalMesh>("hero_mesh");
  • ✅ 支持热更 — 映射文件可随热更包下发
  • ✅ 支持多包合并 — MergeAssetNameMappingByPathAndKey

② FlatBuffers + MMap 极致性能

提供三套查询后端，按平台自动选择最优方案：

模式实现适用场景
普通模式TMap<FString, FString>通用场景
Ansi 模式TMap<FAnsiStringView, ...>减少 Unicode 内存开销
FlatBuffers MMap内存映射文件 + 加速索引iOS/Mac，零拷贝，OS 管理内存
C++ // iOS/Mac 默认开启 MMap 模式 static int32 GEnableAssetNameMappingMMap = 1; // iOS/Mac static int32 GEnableAssetNameMappingMMap = 0; // Android/PC

③ 流式读取 — 内存友好

4KB 分块流式读取映射文件，避免大文件一次性加载到内存，将峰值内存压到最低。

C++ constexpr int32 BufferSize = 4 * 1024; TArray<uint8> Buffer; Buffer.SetNumUninitialized(BufferSize); // 逐块读取并解析...

④ 内置性能分析埋点

所有同步/异步加载接口都内置了性能采集代理，可实时监控每个资产的加载耗时。

C++ // 每次加载都会广播性能数据 if(UMoeAssetLoadAnalysisSubsystem:: OnMoeProfilerCollectAssetLoading.IsBound()) OnMoeProfilerCollectAssetLoading.Broadcast( StartTime, StartSeconds, {ObjectPath}, false);

⑤ 线程安全的加载失败上报

配合 Pak 错误事件监听，精准定位线上资产缺失问题。

C++ // 监听 Pak 错误事件，收集失败资产名 FCoreDelegates::OnAssetLoadingReport.AddUObject( this, &UMoeAssetManager::OnAssetLoadingReport); // 线程安全的失败列表 static TArray<FString> FailedAssetNames; static FCriticalSection FailedAssetNamesCriticalSection;

⑥ UI 资源专项支持

针对 UI 系统提供颜色表、字体表、DataTable 快捷访问接口，并带有缓存。

C++ FLinearColor GetUserWidgetColor(const FString& ColorName); FSlateFontInfo GetUserWidgetFont(const FString& FontName); UDataTable* GetUserWidgetTable(const FString& Name);

❓ 为什么不直接用 UAssetManager？

UAssetManager 的设计假设与手游场景存在本质冲突。

UAssetManager 的设计假设

资产在编辑器中被扫描注册
ScanPathsForPrimaryAssets 需要 AssetRegistry 全量扫描，手游场景启动慢
资产通过 PrimaryAssetId 寻址
要求资产实现 GetPrimaryAssetId() 接口，增加接入成本
Bundle/Chunk 管理面向 PC/主机
面向大型 DLC 分包，不适合手游的热更模型
全局单例模式
无法支持多个独立的资产命名空间（不同业务模块的映射文件隔离）

元梦之星的核心挑战

🔥 热更频繁

资产路径随版本变化，需要映射文件动态下发

🧠 内存敏感

手机内存有限，AssetRegistry 全量缓存代价太高

⚡ 启动性能

UAssetManager 的初始扫描在手游上会显著拖慢启动速度

📝 短名习惯

策划/程序习惯用短名配置资产，而非完整路径

💡
总结：UMoeAssetManager 选择绕过 UAssetManager 的 PrimaryAsset 体系，直接封装 FStreamableManager（UAssetManager 内部也是用它加载的），自建了一套更轻量、更适合手游热更场景的短名映射系统。
