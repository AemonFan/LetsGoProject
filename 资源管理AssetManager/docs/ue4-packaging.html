📊 FPakOrderMap — 加载顺序优化

管理 Pak 文件中文件的顺序索引，优化资源加载性能。

sequenceDiagram
    participant Client
    participant FPakOrderMap

    Client->>FPakOrderMap: AddOffset("Texture.uasset", 100)
    FPakOrderMap->>OrderMap: 存储偏移量映射
    Client->>FPakOrderMap: AddOffset("Mesh.uasset", 200)

💡
通过顺序优化，频繁访问的资源可以放置在 Pak 文件开头，减少磁盘寻址时间，提升游戏启动和运行时的资源加载性能。

🍳 Cook 资源格式转换

Cook 系统负责将原始资源转换为平台特定格式。支持 CookOnTheFly（实时）和 CookByTheBook（批量）两种模式。

classDiagram

        -ECookMode CurrentCookMode
        -FCookByTheBookOptions CookByTheBookOptions
        -FCookFlags CookFlags

        +GenerateAssetRegistry()
    }
    class FPlatformManager {
        -TArray PlatformDatas

        +ReleaseCookOnTheFlyPlatform()
    }
    class FCookCommandlet {
        -bool bIterativeCooking
        -bool bCookOnTheFly

        +CookByTheBook()
        +Main() int32
    }
    UCookOnTheFlyServer --> FPlatformManager
    FCookCommandlet --> UCookOnTheFlyServer

CookOnTheFly 实时烹饪

用于编辑器实时烹饪，按需处理请求队列。RequestQueue 遵循 FIFO 原则，LoadReadyQueue 按依赖关系排序。

CookByTheBook 批量烹饪

用于大规模批量烹饪，支持 bIterativeCooking（增量烹饪）和 bCookAll（全量烹饪）选项。

压缩算法

命令行参数指定：-compression=Oodle、-compression=LZ4。IoStore 通过 SetUseIoStore(true) 启用。

加密与签名

通过 FEncryptionKeyCache 类管理加密密钥，GetPakSignatureFile 进行签名验证。控制台变量 GPakCache_Enable 启用缓存。
