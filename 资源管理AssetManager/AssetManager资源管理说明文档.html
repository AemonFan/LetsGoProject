资源分包策略

元梦之星项目庞大、玩法多样，每个玩法可分为多个资源包，支持按需下载和加载。

📱 基础包（小包）

包含核心玩法资源，下载即可开始游戏。如狼人玩法的基础 UI、角色模型、核心逻辑资源。

💎 资源包（大包）

包含商业化资源、高清材质、额外皮肤等，需额外下载才能使用。

生命周期全链路

① 下载（DownloadChunks）
从 CDN 下载 .pak 文件到本地，支持并发下载、断点续传，以 .pak 文件为最小下载单位
② 挂载（MountChunks）
将 Chunk 对应的 .pak 文件挂载到引擎文件系统，使其中的资源可以被访问
③ 加载映射表（InitAssetNameMappingByPath）
加载对应玩法的 AssetNameMapping 文件，建立短名到长路径的映射关系
④ 加载资源（LoadSyncAssetObject / RequestAsyncLoad）
通过短名查找完整路径，经由 FStreamableManager 加载 UObject 资源到内存
⑤ 使用资源
游戏运行中正常使用已加载的资源
⑥ 卸载资源 → 清除映射
退出玩法时先 UnloadAssetObject() 卸载资源，再 ClearAssetNameMappingByPath() 清除映射表
⑦ GC 回收
引用计数归零后的 UObject 在下次垃圾回收时真正释放内存

🔍 资源加载失败常见原因

项目中总结的资源加载失败典型场景及解决方案。

❌ 原因一：资源本身不存在

  • 资源未上传至引擎
  • CDN 资源未上传至 CDN 或 CDN 资源路径不正确

🔗 原因二：资源引用丢失

  • UI 蓝图中直接引用资源，资源路径改动可能导致资源丢失，需重新引用并编译 UI 蓝图
  • UI 蓝图中引用了其他目录的资源，在手机端分包后，其他目录的资源如果没有加载，则当前引用会丢失

📐 原因三：资源格式不正确

  • 资源过大 — 如资源最大为 5MB、图片大小不超过 1920×1080
  • 资源类型不支持 — 支持动态加载的 UI 资源类型：astc / png / jpg / map
  • ASTC 格式约束 — astc 压缩格式的图片宽度和高度必须是块大小的整数倍
📋
真实案例：astc 块大小为 8×8 像素，图片尺寸为 408×420 像素。408 ÷ 8 = 51（✅ 整数），420 ÷ 8 = 52.5（❌ 不是整数）→ 高度不符合要求，图片下载失败。

🗺️ 原因四：AssetNameMapping 映射文件相关

  • 动态加载的资源必须导出 AssetNameMapping 文件，否则无法从映射表中找到全路径
  • 部分类型的资源需要在 AssetDirectorySetTable_NR3E 表中添加导出路径
  • AssetNameMapping 按玩法需求加载，退出玩法后对应文件会卸载，在此玩法外引用玩法资源会导致加载失败
🎮
真实案例：主玩法需要在祈愿活动中预览狼人的局内道具"会议称号"。会议称号的预览中部分 UI 资源在狼人分包中，局外只有在狼人备战场景才会加载 AssetNameMapping_NR3E.txt 文件，退出后卸载，导致祈愿预览场景无法找到狼人的 UI 资源而报错。
