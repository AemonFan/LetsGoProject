3.2 详细功能拆解

1️⃣ 阶段1：技能可用性判断

在 NR3E3_VocationAbility_CheckWolf.lua 中：

函数作用限制条件
K2_CanActivateAbility()GAS框架级技能可用性判断必须在讨论/投票阶段，且不能查验队友
GetBtnShow()UI按钮是否显示目标不能是自己、技能者存活、目标存活、拥有CheckWolf标签
GetBtnIndex()UI按钮状态(0=可用/1=置灰)每会议1次限制、总次数≤2次、不能查验队友、已查验过的置灰
次数限制机制（基于GAS Cost Tag）：
• NR3E_E3_Cost_MaximumTimes_OneMeetingCheckWolf — 单次会议可查验次数（每次进会议通过 SetOneMeetingCount() 重置为1）
• NR3E_E3_Cost_MaximumTimes_CheckWolf — 全局剩余可查验次数

2️⃣ 阶段2：UI交互 — 二次确认弹框

UI_InLevel_NR3E3_MettingCheckWolf.lua 承担二次确认功能：

  • 打开时机：OnClickedBtnCheckWolf() 中通过 _MOE.UIManager:OpenWindow() 打开
  • 传入数据：SureCallBack（确认回调）、ShowSelectUID（目标UID）、Title（"是否查验X号"）、Tips（剩余次数提示）
  • 确认操作：OnClickedBtnSure() 执行 SureCallBack(ShowSelectUID, ShowSelectIndex) → 调用 CheckWolf()
  • 自动关闭：监听 MeetingSettlement 事件，会议结算时自动关闭弹框

3️⃣ 阶段3：技能执行 — 客户端到服务器

CheckWolf() 函数的关键流程：

1. 构造 EffectContext，携带 TargetUID
2. SendGameplayEventToActor → 通过 GameplayTag 触发服务器端技能激活
3. 服务器端 K2_ActivateAbilityFromEvent() 执行核心逻辑

4️⃣ 阶段4：服务器权威判定

K2_ActivateAbilityFromEvent() 是核心的服务器权威逻辑：

  1. 合法性校验：不能自查、目标信息存在、目标未死亡
  2. 请求职业信息：UMoeNR3EGameLibrary.RequestVocationInfoSpecificPlayer() 获取目标真实阵营
  3. 判定结果：playerInfo:GetCamp() == UE4.EMoeNR3EVocationCamp.Hunter → 狼人；否则→非狼人
  4. 通知客户端文字结果：通过 NR3E_Server2Client 事件机制分发
  5. 应用GameplayEffect：ApplyGameplayEffectSpecTo() 在目标身上添加查验标记Buff
  6. 记录关键事件：K2_DispatchAbilityEvent 记录预言家查验的重要事件日志

5️⃣ 阶段5：GameplayCue — HUD头顶标记显示

NR3E3_VocationCue_CheckWolf.lua 是查验结果的视觉表现层：

可见性控制：
CanActiveGameplayCue() — 仅当本地玩家是技能释放者（预言家本人）时才激活，确保只有预言家能看到标记

HUD显示逻辑（SetCueVisible()）：

  • 区分 傀儡(Puppet) 和 真实角色 获取不同的组件
  • 判断目标阵营：Hunter → 狼人标记(参数1) / 非狼人 → 好人标记(参数4)
  • 调用 Component:SetBillBoardFactionShow(true/false, factionId) 设置头顶Billboard标识

特殊场景处理：

事件处理函数说明
ON_NR3E3_VOCATION_SETOnActorChangeIndetify()配合伪装者身份变更，实时刷新查验结果
ON_NR3E3_DeathEvent_WhileActive_ClientOnActorDead()预言家死亡时隐藏查验标记
ON_NR3E3_DeathEvent_OnRemove_ClientOnActorResurrect()预言家复活时重新显示查验标记
⚡ 伪装者联动：当被查验目标（伪装者）修改身份时，OnActorChangeIndetify() 会使用新的 VocationCamp 重新调用 SetCueVisible()，实现查验结果的实时变化。

🔧四、技术点总结

4.1 GAS (Gameplay Ability System) 应用

GAS组件对应资产用途
GameplayAbilityVocationAbility_CheckWolf技能激活/释放/取消/结束的生命周期管理
GameplayEffectVocationEffect_CheckWolf作为查验标记Buff施加到目标身上，触发GameplayCue
GameplayCueVocationCue_CheckWolfResult查验结果的视觉表现（头顶HUD标记）
GameplayTag多个Tag技能触发标识、次数Cost限制、属性传递
EffectContext自定义属性携带 InstigatorUID、TargetUID、isWolf 等自定义数据

4.2 网络同步机制

graph LR subgraph "客户端(预言家)" A[玩家操作] --> B[构造EffectContext] B --> C[SendGameplayEvent] end subgraph "服务器(权威端)" C --> D[K2_ActivateAbilityFromEvent] D --> E[判定阵营结果] E --> F[ApplyEffect到目标] E --> G["NR3E_Server2Client事件"] end subgraph "客户端(同步回来)" F --> H[GameplayCue触发] H --> I["CanActive过滤(仅预言家可见)"] I --> J["SetBillBoardFactionShow(显示头顶标记)"] G --> K["HandleCheckWolfResult(文字弹窗提示)"] end

关键同步设计：

  1. GameplayEvent 触发技能：客户端通过 SendGameplayEventToActorWithParameters() 将操作发送到服务器
  2. 服务器权威判定：K2_IsAuthoirty() 保证阵营判定只在服务器执行，防止客户端作弊
  3. GameplayEffect 网络同步：Effect 自动复制到所有客户端，触发各客户端的 GameplayCue
  4. GameplayCue 本地过滤：CanActiveGameplayCue() 通过 UID 比对，确保只有预言家本人看到标记
  5. 自定义Server2Client事件：NR3E_Server2Client 事件将查验文字结果通知到指定客户端

4.3 HUD显示技术

  • Billboard组件：通过 SetBillBoardFactionShow(isShow, factionId) 控制玩家头顶的阵营标识
    • factionId = 1：狼人标记（红色）
    • factionId = 4：好人标记（蓝色）
  • Puppet兼容：区分真实角色和傀儡（Puppet），获取不同的组件来显示标记
  • 观察者支持：IsLocalPlayerObserverAndLookInstigator() 支持观战视角也能看到查验标记

4.4 生命周期管理

阶段回调函数行为
技能添加K2_OnAbilityAdded注册查验结果事件监听
技能移除K2_OnAbilityRemoved预言家死亡后，遍历 BuffTargetList 清除所有已施加的查验 Effect
Cue激活K2_OnActive注册身份变更、死亡、复活事件
Cue移除K2_OnRemove隐藏标记

✨五、设计亮点

  1. 伪装者联动：查验结果不是静态的，会随伪装者身份切换而实时变化
  2. 防作弊设计：阵营判定在服务器权威端执行，客户端仅负责表现
  3. 次数管控：通过 GAS 的 Cost Tag 机制，分别控制单次会议和全局查验次数
  4. 灰态提示：按钮不可用时，给出具体原因（已查验/次数耗尽/队友/非会议阶段）
  5. 观战兼容：支持观战视角查看预言家的查验标记结果
  6. 生命周期完善：预言家死亡/复活时正确处理HUD标记的显示/隐藏

🗺️六、总结流程一览

玩家点击查验 → GetBtnShow/GetBtnIndex判断可用性
    → OnClickedBtnCheckWolf() 打开二次确认弹框
    → 用户点击确认 → CheckWolf()
    → 构造EffectContext + SendGameplayEvent → 服务器端
    → 服务器校验合法性 + 请求目标阵营 → 判定是否狼人
    → ApplyGameplayEffect到目标 → 触发GameplayCue
    → CanActiveGameplayCue过滤(仅预言家可见) → SetBillBoardFactionShow显示头顶标记
    → Server2Client通知文字结果 → HandleCheckWolfResult弹出提示
