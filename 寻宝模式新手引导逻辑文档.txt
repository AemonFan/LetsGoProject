一、新手引导系统概述
================================================================================

寻宝模式的新手引导基于通用的 PlayerGuideModel 实现：
  - _MOE.Models.PlayerGuideModel:HasFinishGuide(guideID)  -- 判断引导是否已完成
  - _MOE.Models.PlayerGuideModel:EndCurrentGuide(true)     -- 结束当前引导（标记完成）
  - _MOE.Models.PlayerGuideModel:GetCurrentGuideID()       -- 获取当前正在进行的引导ID

核心逻辑：
  1. 通过 HasFinishGuide(guideID) 判断玩家是否已完成某个引导步骤
  2. 如果未完成，则通过 DispatchEvent 派发引导事件名称
  3. 事件名称对应新手引导配置表中的 "触发事件"，引导系统监听该事件后展示引导UI
  4. 引导分为 平民(Prey)/狼人(Hunter) 两个阵营独立配置，同一时机有两套引导ID

PlayerGuideModel 关键方法说明：
  定义在 LetsGo/Content/LetsGo/Script/Model/Guide/PlayerGuideModel.lua
  - HasFinishGuide(guideID)：先检查AB测试分组，再检查本地缓存和服务端已完成列表
  - IsGuideAvailable(guideID)：判断引导是否可用（未完成且满足前置条件）
  - StartGuide(guideID)：开始某个引导
  - EndCurrentGuide(withComplete)：结束当前引导，withComplete=true表示标记完成
  - GetCurrentGuideID()：获取当前正在进行的引导ID

触发时机：游戏开始时的"开始界面"（StartView）关闭后
触发条件：HasFinishGuide(503052/503060) == false

代码位置：
  UI_InLevel_NR3E3_TreasureHunt_StartView.lua -> OnClose()
  路径：G:/LetsGoDevelop/LetsGo/Content/Feature/NR3E/Script/System/InLevel/

      -- ... 清理定时器 ...
      if not _MOE.Models.PlayerGuideModel:HasFinishGuide(503052) then
          _MOE.EventManager:DispatchEvent("ON_NR3E3TreasureHuntRule_Prey_StartGameGuide")

  - 平民阵营和狼人阵营分别触发各自的引导

代码逻辑（触发引导 - 按钮高亮时）：

      if _MOE.GEnv.IsPCStyle then
          -- PC端引导
          if not HasFinishGuide(503070) then DispatchEvent("Prey_DiggingTreasureGuide_PC")
          if not HasFinishGuide(503072) then DispatchEvent("Hunter_DiggingTreasureGuide_PC")
      else
          -- 移动端引导
          if not HasFinishGuide(503053) then DispatchEvent("Prey_DiggingTreasureGuide")
          if not HasFinishGuide(503061) then DispatchEvent("Hunter_DiggingTreasureGuide")
      end
  end

PC端特殊处理（通过快捷键直接完成引导）：
  -- 在 K2_ActivateAbility 中
  if _MOE.GEnv.IsPCStyle and not self:K2_IsAuthoirty() then
      local guideId = _MOE.Models.PlayerGuideModel:GetCurrentGuideID()
      if guideId == 503070 and not HasFinishGuide(503070) then
          _MOE.Models.PlayerGuideModel:EndCurrentGuide(true)
      end
      if guideId == 503072 and not HasFinishGuide(503072) then
          _MOE.Models.PlayerGuideModel:EndCurrentGuide(true)
      end
  end

五、教学提示系统（TeachingTips）—— 另一种引导方式
================================================================================

除了新手引导配置表驱动的引导外，寻宝模式还有一套"教学提示"（TeachingTips）系统，
基于 NR3E3TipsModel（位于 NR3E/Script/System/E3/Models/NR3E3TipsModel.lua）实现。

寻宝模式相关的 TeachingTipsType 枚举定义（在 NR3E3GameConfig.lua 第1314行起）：

  TeachingTipsType.TreasureProgressReach   = 1000  -- 达到寻宝进度
  TeachingTipsType.TreasureCarryDelay      = 1001  -- 搬运宝物3秒
  TeachingTipsType.TreasureMeetingEndToMain = 1002 -- 会议结束回到主界面(寻宝)
  TeachingTipsType.TreasureAttack          = 1003  -- 摧毁宝物

各触发点说明：

【TreasureProgressReach (1000) - 寻宝进度达成提示】
  触发位置：UI_InLevel_NR3E3_TreasureHunt_Task.lua -> UpdateUI()
  触发时机：寻宝进度条动画播放完成后（alpha >= 1.0）
  代码：
      _MOE.EventManager:DispatchEvent(NR3E3GameConfig.EventEnum.E3TeachingTips, {
          Type = NR3E3GameConfig.TeachingTipsType.TreasureProgressReach,
          Params = {
              totalProgress = self.TreasureHunt_TotalProgress,
              curProgress = math.floor(currentProgress)
          }
      })
  检查逻辑：CheckTaskProgressReach（复用普通任务进度检查）

【TreasureCarryDelay (1001) - 搬运宝物3秒提示】
  触发位置：NR3E3_VocationAbility_CTH_TreasureDrop_New.lua（第56行）
  触发时机：玩家搬运宝物持续3秒后
  代码：
      _MOE.EventManager:DispatchEvent(NR3E3GameConfig.EventEnum.E3TeachingTips, {
          Type = NR3E3GameConfig.TeachingTipsType.TreasureCarryDelay
      })
  检查逻辑：CheckDefault（默认通过）

【TreasureMeetingEndToMain (1002) - 寻宝会议结束回到主界面提示】
  触发位置：由游戏流程自动触发
  检查逻辑：CheckTreasureMeetingEndToMain
      - conditionParams[1] == 0 -> 未搬运宝物时触发
      - conditionParams[1] == 1 -> 正在搬运宝物时触发

【TreasureAttack (1003) - 摧毁宝物提示】
  触发位置：NR3E3AbilityActor_TreasureBase.lua（第66行）
  触发时机：服务端->客户端 通知宝物被摧毁时
  代码：
      _MOE.EventManager:DispatchEvent("NR3E_Server2Client",
          NR3E3GameConfig.EventEnum.E3TeachingTipsS2C,
          Instigator.UID,
          NR3E3GameConfig.TeachingTipsType.TreasureAttack,
          Instigator.UID)
  检查逻辑：CheckDefault（默认通过）
