1. 系统概览

商业动画系统 (Commercial Animation System) 位于 Commercial 目录下，是狼人杀游戏中负责管理各类过场动画的核心子系统。系统采用 UE4 Level Sequence + Streaming Level 技术方案，实现了击杀动画、会议报告动画、投票淘汰动画、MVP动画等多种场景类型的加载、播放、切换与销毁。


支持的场景类型：

击杀场景 DeathScene 会议报告 MeetingReport 投票淘汰 MeetingOut MVP展示 MVP 商店场景 Shop 商店轮播 ShopCircle 寻宝PV TreasureHuntPV

4. 生命周期管理

4.1 基类模板方法

基类 BP_NR3E3CommercialScenePlayAction_Base 定义了完整的动画生命周期模板：

方法阶段说明
CanActive(SceneId)激活检查判断当前场景是否可以激活
Init()初始化注册事件、加载资源、初始化状态
OnStartLoadMap(LevelPath)加载开始关卡开始流式加载
OnMapLoadFinish(LevelInfo)加载完成关卡加载完毕，开始准备动画
OnStartPlayLevelSequence序列准备Level Sequence即将开始播放
OnLevelSequencePlayStart序列播放Level Sequence开始播放
OnLevelSequencePlayFinish序列结束Level Sequence播放完毕
PreUnLoadLevel()卸载准备关卡卸载前的清理工作
OnMapHide(LevelInfo)隐藏关卡隐藏
OnMapUnLoad(LevelInfo)卸载关卡卸载完成
UnInit()反初始化清理事件、定时器、恢复状态

4.2 淡入效果与前后台切换

基类支持可选的 FadeIn 淡入效果（通过 WaitForFadeIn 控制），以及前后台切换事件处理：

  • 开启淡入时：先显示UI遮罩，延迟0.2秒后执行加载完成逻辑
  • 通过 CanExecuteLoadFinishAction() 判断淡入完成 + LevelInfo有效后才继续
  • 注册 ON_APPLICATION_ENTER_BACKGROUND/FOREGROUND 事件处理前后台切换

5. 动画加载与播放机制

5.1 Level Sequence 驱动

系统使用 UE4 的 Level Sequence 作为核心动画驱动方式：

-- 序列播放开始时，先暂停序列，等待角色动画准备
function MVP:OnLevelSequencePlayStart(SequencePlayerActor)
    local SequencePlayer = SequencePlayerActor:GetSequencePlayer()
    SequencePlayer:Pause()   -- 先暂停

    -- 0.5秒后播放角色骨骼动画
    _MOE.TimerManager:AddTimer(0.5, self, function()
        _MOE.NR3E3_CommonFunc.Animation:PlayMVPAnimations(...)
    end)
end

-- 加载完成退出后，恢复序列播放
function MVP:ResetPlayLevelSequence()
    SequencePlayer:Play()    -- 恢复播放
end

5.2 骨骼动画序列 (AnimSequence)

配置字段用途说明
MVPAnimSequence单人MVP动画标准2/2.5头身角色
TwoPlayerMVPAnimSequence双人MVP动画标准2/2.5头身角色
MVPAnimSequence35单人3.5头身动画3.5头身特殊角色专用
TwoPlayerMVPAnimSequence35双人3.5头身动画3.5头身特殊角色专用
MVPAnimSequenceSpecific特定角色单人动画按角色ID定制，如黄油小熊[412230]
TwoPlayerMVPAnimSequenceSpecific特定角色双人动画按角色ID定制
OtherAnimSequence10P/12P/15P其他玩家动画按人数配置不同分排动画

5.3 角色头身比适配

标准体型

  • 2头身 - 使用标准 MVPAnimSequence
  • 2.5头身 - 使用标准 MVPAnimSequence

特殊体型

  • 3.5头身 - 使用专用 MVPAnimSequence35（后缀_TF）
  • 特定角色 - 使用 MVPAnimSequenceSpecific（如Bear）

动画匹配优先级：Specific(角色ID定制) > 35(3.5头身) > 标准动画

6. 相机系统

6.1 相机控制组件

MoeNR3E3CameraControlComponent 负责相机的视角切换：

  • SetCameraViewTarget(Type, MapName, BlendTime) - 设置相机目标并平滑过渡
  • RestViewTargetToDefaultCamera(BlendTime) - 恢复默认相机视角
  • 通过 SetViewTargetWithBlend 实现带混合时间的平滑过渡

6.2 屏幕适配器

NR3ECommercialSceneCameraAdapter 根据屏幕比例动态调整相机距离：

-- 配置范围: 16:9 ~ 21:9
-- 线性插值计算调整距离
AdjustDistance = MinDist + (MaxDist - MinDist)
    * (viewportRatio - MinRatio) / (MaxRatio - MinRatio)

确保在不同设备（手机、平板、PC）上都能获得合适的画面构图。

7. 角色系统

7.1 角色更新流程 (UpdateMVPAvatar)

  1. 数据校验 - 检查 CallData、MVPAvatarInfo、AvatarCharacterList 是否有效
  2. 定位MVP角色 - 通过UID匹配找到旧MVP Actor
  3. 获取新位置 - 从Level中获取CharacterType=1的新Actor位置
  4. 隐藏所有新Actor - 先将Level中15个角色位全部隐藏
  5. 逐个分配位置 - 遍历AvatarInfoList，将旧角色移动到新位置
  6. 模型替换 - 根据变身缓存或原始装扮替换模型
  7. LOD控制 - 非主要角色强制低LOD降低性能开销
  8. 高度修正 - 调用FixHeight修正多排角色的高度对齐

7.2 变身缓存机制

-- 检查变身缓存
local ModelIndex = _MOE.NR3E3InLevelModel.MVPModelCache[AvatarInfo.uid]
if ModelIndex then
    self:ReplaceCacheAvatar(NewActor, ModelIndex)  -- 有缓存：直接替换骨骼网格体
else
    Model:ReplaceAvatarByAvatarInfoList(...)       -- 无缓存：通过装扮信息异步加载
end

ReplaceCacheAvatar 核心：隐藏 MasterSkelMeshComp，将 SkinSkelMeshComp 替换为缓存的骨骼网格体。

7.3 多排高度修正与LOD控制

  • FixHeight: 按Row分组(1/2/3排)，分别校准Widget组件高度对齐
  • LOD策略: 单人模式仅主角高LOD；双人模式前2位高LOD，其余低LOD

8. 资源管理与预加载

8.1 预加载系统 (InLevelPreLoader)

采用 Server收集 + Client差量更新 的模式：

Server端 (DS)：

  1. 游戏开始时收集所有玩家的报告装备场景ID
  2. 监听职业分配完成事件，收集攻击装备场景ID
  3. 场景替换时，清理并重新下发预加载ID列表
  4. 通过 PreLoadIds 属性同步到Client

Client端：

  1. OnRep_PreLoadIds 接收Server同步的ID列表
  2. 与已加载列表对比，计算差量（ToAddList / ToRemoveList）
  3. 新增场景立即预加载（RequestPreLoadMap）
  4. 移除场景延迟卸载，避免动画被中断（会议2s/其他15s）

8.2 Streaming Level 流式加载

MoeNR3E3StreamingLevelComponent 封装了UE4的Level Streaming机制：

  • Receive_OnLoadStreamLevelFinish - 加载完成后派发事件
  • Receive_ShowLevel - 显示关卡后派发事件
  • Receive_UnloadLevel - 卸载关卡后派发事件

11. 核心流程

11.1 MVP动画完整播放流程

1. Init()
   ├─ 注册事件(场景切换/加载退出)
   ├─ 隐藏反射球, 注册特殊UI窗口
   └─ 预创建AvatarPreviewActor

2. OnMapLoadFinish(LevelInfo)
   ├─ 调用基类处理, 开启场景灯光
   ├─ 设置场景相机
   ├─ UpdateMVPAvatar() -> 角色换装/定位
   ├─ 判断MVPUID显隐角色
   └─ UpdateCamera() -> 相机偏移

3. OnLevelSequencePlayStart()
   ├─ 暂停Level Sequence
   ├─ 收集场景内角色Actor
   └─ 0.5秒后播放骨骼动画

4. OnEnterLevelLoadingOut()
   └─ 0.5秒后恢复Level Sequence播放

5. OnLevelSequencePlayFinish()
   ├─ 调用基类清理多播事件
   └─ 重新播放MVP循环动画

6. Finish()
   ├─ 调用UnInit()清理
   └─ 1秒后请求结束预览动作

11.2 预加载流程

Server端:
  1. 游戏开始 -> 收集报告装备场景ID
  2. 职业分配 -> 收集攻击装备场景ID
  3. 场景替换 -> 全量刷新预加载列表
  4. PreLoadIds属性同步到Client

Client端:
  1. OnRep_PreLoadIds -> 接收ID列表
  2. 差量计算 -> ToAddList / ToRemoveList
  3. 新增项 -> 立即预加载
  4. 移除项 -> 延迟卸载(会议2s/其他15s)

12. 系统优点

✔ 良好的继承体系

Base基类定义完整的生命周期模板方法，子类按需重写，代码复用度高。

✔ 数据驱动配置

13套MVP动画配置独立管理，支持单人/双人/多人，且能针对特定角色ID定制动画。

✔ 完善的预加载机制

Server收集、Client差量更新减少IO，延迟卸载避免动画中断。

✔ 多体型适配

为2头身、2.5头身、3.5头身及特殊角色提供完整的动画适配方案。

✔ 事件解耦架构

组件间通过事件通信，降低耦合度。

✔ 多设备屏幕适配

CameraAdapter在16:9到21:9范围内自动调整相机距离。

13. 系统不足与改进建议

⚠ 核心Model文件过大

NR3ECommercialPreviewModel.lua 高达 68KB。建议按功能拆分为多个子Model。

⚠ 魔法数字较多

硬编码 3000000、25、15、241012 等。建议提取为命名常量。

⚠ 调试日志未清理

大量 kfc1 前缀调试日志残留。建议统一日志前缀并分级控制。

⚠ 定时器依赖过重

多处使用固定时长定时器(0.5s~15s)衔接流程。建议改用回调机制。

⚠ 部分同步加载

UE4.LoadObject 为同步加载，可能导致卡顿。建议改用异步加载。

⚠ 注释不规范

部分函数缺少注释，中英文混杂。建议统一注释规范。

14. 总结

狼人杀商业动画系统是一套基于 UE4 Level Sequence + Streaming Level 的成熟过场动画解决方案。通过良好的继承体系、事件驱动架构和数据驱动配置，实现了多种场景类型的统一管理。系统特别在多体型角色适配（2头身/2.5头身/3.5头身）、预加载机制、变身缓存等方面表现出色。


核心技术栈：

UE4 Level Sequence Streaming Level Lua (UnLua) AnimSequence Chunk Pak 事件驱动 LOD优化 异步加载

后续建议重点关注：核心Model拆分、魔法数字消除、定时器替换为回调机制、同步加载改异步，以进一步提升系统的可维护性和运行性能。
