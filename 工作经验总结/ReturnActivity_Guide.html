📋 一、系统概述

狼人回归系统（ReturnActivity）是为「狼人杀」玩法中的回归玩家量身设计的限时活动系统。其核心目标是通过专属奖励机制吸引流失玩家回归，提升用户留存率。

🎯 核心设计要点：
● 触发条件：回归玩家登录游戏后自动弹出活动弹窗
● 奖励机制：三档递进式奖励，完成对局后逐步解锁
● 技术栈：UE4 + UnLua + Lua MVC 架构

📌 3.1 回归欢迎弹窗（PopupView）

文件：UI_Preparations_ReturnActivity_PopupView.lua

功能概述

回归玩家登录后弹出的全屏欢迎界面，展示活动说明并引导玩家进入活动。

界面元素

  • 活动名称与图标
  • 5条活动说明文案（w_txt_Tip1 ~ w_txt_Tip5）
  • "前往领取" 按钮 —— 播放飞行动画后跳转至主界面
  • "查看公告" 按钮 —— 打开狼村公告界面

核心交互 — 飞行动画流程

  1. 用户点击 "前往领取"
  2. 播放缩小动画（UMG_small）
  3. 隐藏模糊背景层
  4. 延迟 0.3s 后启动贝塞尔曲线飞行动画
  5. 图标沿弧线从弹窗中心飞向活动入口按钮位置
  6. 飞行结束 → 关闭弹窗 → 刷新活动入口显示

技术亮点

二次贝塞尔曲线插值：B(t) = (1-t)²·P0 + 2t(1-t)·P1 + t²·P2
跨界面坐标转换：ComputeRelativeLocalPos 统一坐标空间
目标Widget获取：GetTargetWidget 通过 UIManager 获取目标界面 Widget 引用

📌 3.2 回归活动主界面（MainView）

文件：UI_Preparations_ReturnActivity_MainView.lua

功能概述

活动核心界面，展示三档回归奖励的总览信息及领取状态。

界面布局

  • 顶部：活动时间（open_ts ~ end_ts 格式化显示）
  • 中部：三个奖励卡片
    • 奖励1（w_RewardItem_1）—— 限时报告动画
    • 奖励2（w_RewardItem_2）—— 限时攻击动画
    • 奖励3（w_RewardItem_3）—— 永久狼人身份（大尺寸卡片）
  • 每个卡片含进度描述文字、状态按钮

奖励状态机

state = -1
未解锁
初始态，显示"待领取"
➡️
state = 0
已解锁
完成对局，显示进度提示
➡️
state = 1
可领取
满足条件，显示"领取"
➡️
state = 2
已领取
播放 UMG_OK 勾选动画

使用 WidgetSwitcher（w_switcher_BtnState）根据状态切换按钮显示。

核心交互

  • 点击 "解锁态" 按钮（w_btn_UnLock）→ 打开奖励预览界面
  • 点击 "可领取态" 按钮（w_btn_Locked）→ 打开领取操作界面
  • 奖励3的领取标题文案支持自定义替换
  • 领取前校验分包下载状态

事件监听

📌 3.3 奖励预览与领取界面（RewardView）

文件：UI_Preparations_ReturnActivity_RewardView.lua

功能概述

用于预览和领取第1档（报告动画）、第2档（攻击动画）奖励的二级界面。

双模式设计

👁 预览模式

isOperate = false

仅展示奖励内容，不可操作

✏️ 领取模式

isOperate = true

展示选择与确认按钮，可操作领取

数据处理流程

  1. 从 select_list 解析奖励物品 ID 和数量
  2. 通过 BagModel:GetItemConfigById 获取物品配置
  3. 判断物品是否已拥有（maxNum==1 时检查背包）
  4. 通过 WolfKillComeBackTable 获取物品过期天数
  5. 区分物品类型：动画类 / 其他类

排序规则

档位排序优先级
第1/2档动画类 > 其他类 > 已拥有 > 未拥有，同类按 ID 升序
第3档身份类 > 其他类，同类按 ID 升序

选择与领取流程

  1. 在 UListView（w_list_Reward）中选择奖励（最多 MaxPickNum = 1 个）
  2. 选满 → 显示确认按钮  |  未选满 → 显示禁用按钮
  3. 确认后：
    • 更新红点状态（E_RDT_TYPE.PlayerReturn）
    • 调用 PreparationsModel:GetReturnActivityReward 发送领取请求
    • 关闭当前界面

事件监听

事件名称说明
BagModel_PackPick_Changed物品选择变更
BagModel_BagUsePropItemRsp使用物品响应
BagModel_ItemPackagePickPickedNumRsp已领取数量响应
PreparationsModel_CloseAllReturnActivityView关闭全部回归界面

数据埋点

埋点ID触发动作附加参数
231023打开奖励预览index，物品ID列表

🔧 四、关键技术点

🏗️ 1. MVC 架构模式

  • Model层：PreparationsModel 管理活动数据（信息、状态、奖励等）
  • View层：UI_ 文件继承 BaseViewClass，负责界面渲染和交互
  • 通信方式：AddEventListener / DispatchEvent / RemoveEventListener
  • 实现了 Model 与 View 的完全解耦

🎮 2. UE4 + UnLua 技术栈

  • 基于 Unreal Engine 4 的 UMG（Unreal Motion Graphics）UI 框架
  • 通过 UnLua 插件实现 Lua 与 UE4 Widget 的绑定
  • LuaDoc 注解声明蓝图绑定：---@field bp XXX_BP
  • 自动生成区域（Auto Generate Begin/End）维护 Widget 引用和事件绑定

🔄 3. 类继承与生命周期

  • _MOE.class 实现 Lua 面向对象的类创建和继承
  • 统一生命周期：
    OnOpen → InitData → InitEvent → InitUI → OnUpdate → PreClose → OnClose
  • PickItem 继承 UI_Bag_GiftPackPickItem 基类复用通用选择交互逻辑
  • 所有事件在 InitEvent 注册、OnClose 移除，防止内存泄漏
  • 数据引用在 OnClose 置 nil，辅助 GC 回收

🎨 4. UI 组件技术

  • WidgetSwitcher：同一位置切换多种状态显示（按钮的启用/禁用态切换）
  • UListView：数据驱动的高性能列表渲染，通过 ListSetDatas 绑定数据
  • UMG 动画系统：AnimationPlayForward / Reverse 控制正播/倒播
    • UMG_OK（已领取勾选）
    • UMG_small（缩小飞行前置）
    • 按钮按下/释放动画通过 OnPressed / OnReleased 委托实现

🎯 5. 贝塞尔曲线飞行动画

  • 类型：二次贝塞尔曲线
  • 公式：B(t) = (1-t)²·P0 + 2t(1-t)·P1 + t²·P2
  • P0 = 起始点（弹窗图标位置）
  • P1 = 控制点（起始和目标中点偏上方，形成弧线）
  • P2 = 目标点（活动入口按钮位置）
  • 在 OnUpdate 中逐帧插值更新位置，LerpDuration 控制飞行时长
  • ComputeRelativeLocalPos 实现跨界面坐标空间统一转换

📖 6. 配置表驱动

  • WolfKillComeBackTable —— 回归活动配置（各档奖励物品ID、过期天数等）
  • NR3E3VocationTable —— 职业配置（身份物品与阵营的映射关系）
  • NR3ETextTable / TextLuaTable —— 多语言文本配置（LocalKey 国际化）

🔴 7. 红点系统集成

  • 领取奖励时调用 RedDotModel:UpdateRedDotByType(E_RDT_TYPE.PlayerReturn)
  • 确保玩家领取后红点自动消除，完成引导行为闭环

📦 8. 分包下载检测

  • 接口：NR3E3PreparationsModel:IsDownLoadPakFinish()
  • 在 MainView 领取操作和 PickItem 动画预览中均有拦截逻辑
  • 参数 true = 弹出下载提示  |  false = 静默检查
