1. 功能概述

在躲猫猫（Hide-and-Seek / NR3E）玩法中，伪装者可以将自身变身为场景中的道具（如箱子、花瓶等），从而躲避搜寻者的追捕。伪装特效系统负责处理角色伪装过程中的视觉表现，包括：

  • 隐身材质替换：伪装者隐身后，将角色皮肤/变身道具的材质替换为特殊的伪装材质（如果冻半透明材质）
  • 双层材质渲染：果冻特效采用两层材质叠加，第二层 Mesh 为原模型的 1.2 倍大小，透明度更低，营造果冻质感
  • 动态形变动画：角色移动时显示动态效果，跳跃时果冻模型根据运动状态进行弹性形变（上升阶段 Z 轴拉伸，下落阶段恢复）
  • 特效管理：管理伪装/消失时的粒子特效、音效播放
  • 材质缓存与恢复：缓存原始材质信息，确保伪装解除后能正确恢复
📌 核心定位：伪装特效是一种装扮道具，对应道具配置表（Items），在配置中可指定隐身材质路径、模型缩放参数、拖尾特效、隐身特效、偏移参数、伪装音效等。

3. 核心功能模块详解（Lua 层）

文件路径：LetsGo/Script/Modplay/InLevel/NR3E1_Changed.lua

3.1 伪装配置获取 — GetDisguiseConfig()

Lua 核心基础方法，所有其他功能均依赖此方法获取道具配置。

① 获取角色 Owner，校验 IsValid 和 UID
↓
② 通过 NR3EInLevelModel:GetPlayerInfoByUID(UID) 获取玩家信息
↓
③ 通过 GetDressUpItemByItemType() 获取类型为 ItemType_NR3E_DisguiseEffect 的装扮道具 ID
↓
④ 通过 Config.Items:GetDataByKey(ItemID) 查询道具配置表，返回完整配置
function NR3E1_Changed:GetDisguiseConfig()
    local characterOwner = self:GetOwner()
    if not UE4.UKismetSystemLibrary.IsValid(characterOwner) or not characterOwner.UID then
        return nil
    end
    if not _MOE.NR3EInLevelModel then return nil end

    local PlayerInfo = _MOE.NR3EInLevelModel:GetPlayerInfoByUID(characterOwner.UID)
    if not PlayerInfo then return nil end

    local ItemID = _MOE.Utils.MoeGameUtils.GetDressUpItemByItemType(
        PlayerInfo:GetPlayerServerData(),
        _MOE.CommonDefine.E_ITEM_TYPE.ItemType_NR3E_DisguiseEffect)
    if not ItemID then return nil end

    local ItemConfig = _MOE.Config.Items:GetDataByKey(ItemID)
    return ItemConfig
end

3.4 Overlay 材质设置与恢复

-- 设置叠加材质：遍历所有 StaticMeshComponent，设置 Overlay Material
function NR3E1_Changed:SetDisguiseOverlayMaterial()
    local overlayMaterial = self:GetInVisibleOverlayMaterial()
    local length = self.StaticMeshComponents:Length()
    for i = 1, length do
        local staticMeshComp = self.StaticMeshComponents:Get(i)
        if staticMeshComp then
            staticMeshComp:SetOverlayMaterial(overlayMaterial)
        end
    end
end

-- 恢复叠加材质：将 Overlay 材质设为 nil，恢复原始外观
function NR3E1_Changed:RecoverDisguiseOverlayMaterial()
    local length = self.StaticMeshComponents:Length()
    for i = 1, length do
        local staticMeshComp = self.StaticMeshComponents:Get(i)
        if staticMeshComp then
            staticMeshComp:SetOverlayMaterial(nil)
        end
    end
end

3.5 复制模型缩放

function NR3E1_Changed:GetDeplicateModelScale()
    local ItemConfig = self:GetDisguiseConfig()
    if not ItemConfig or not ItemConfig.nr3eDecorate
       or not ItemConfig.nr3eDecorate.pack2
       or not ItemConfig.nr3eDecorate.pack2.model
       or not ItemConfig.nr3eDecorate.pack2.model.scale then
        return 1  -- 默认无缩放
    end
    -- 配置值除以100：如 120 -> 1.2 倍
    return tonumber(ItemConfig.nr3eDecorate.pack2.model.scale) / 100
end

3.6 动画参数更新

function NR3E1_Changed:UpdateAnimParams(speed, moveState)
    if self.IsInitAnim then
        local skeletalMesh = self:GetSkeletalMeshLua()
        if skeletalMesh then
            local animInstance = skeletalMesh:GetAnimInstance()
            if animInstance and animInstance.SetSpeedXY
               and animInstance.SetOnGroundWithMoveState then
                animInstance:SetSpeedXY(speed)                    -- 移动速度
                animInstance:SetOnGroundWithMoveState(moveState)  -- 移动状态
            end
        end
    end
end

4. 果冻特效（Jelly Effect）深度分析

果冻特效（道具 ID: 241210）是伪装特效系统中最复杂的一种特效类型，采用双层材质 + 模型复制 + 动态形变的技术方案。

4.1 双层材质机制

层级材质来源应用目标特征
第1层（主体）pack2.materialPath原始 MeshRoot 上的 StaticMeshComponents半透明果冻材质，主体渲染
第2层（叠加）pack3.materialPath通过 SetOverlayMaterial 设置透明度更低的叠加层，增强果冻质感
渲染原理：第1层材质直接替换 StaticMeshComponent 的原始材质槽；第2层通过 UE4 的 SetOverlayMaterial API 叠加渲染在第1层之上。两层材质共同作用产生半透明、有深度感的果冻视觉效果。

4.2 模型复制与缩放（MeshRootDuplicate）

为了增强果冻效果的立体感，系统在原始模型（MeshRoot）下创建了一个复制模型节点（MeshRootDuplicate）：

  • 复制模型的缩放比例通过 GetDeplicateModelScale() 获取，典型值为 1.2 倍（配置值 120 / 100）
  • 复制模型同样应用隐身材质，但由于尺寸更大，形成外层半透明轮廓
  • 蓝图中通过 Update Duplicate Invisible Material 函数为复制模型设置材质

4.3 跳跃形变动画

果冻特效在角色跳跃时会产生弹性形变效果：

跳跃形变流程

角色起跳 → MoveState 变为非 OnGround 状态
↓
上升阶段 → 模型 Z 轴拉伸（纵向变长，横向变窄）
↓
下落阶段 → Z 轴逐渐恢复原始比例
↓
落地 → MoveState 恢复 OnGround，形变结束

形变由 UpdateAnimParams(speed, moveState) 传递运动参数到 UNR3E1ChangedAnimInstance，动画蓝图根据 SpeedXY 和 IsOnGround 状态在 JumpAnim（跳跃动画序列）和 OnGroundBlendSpace1D（地面混合空间）之间切换，驱动 SkeletalMesh 实现弹性形变。

5. 角色伪装成道具 — 完整实现流程

以下详细描述了角色从普通状态变为伪装道具并应用伪装材质和特效的完整流程：

5.1 阶段一：伪装触发

触发条件

① 玩家在躲猫猫模式中选择一个场景道具进行伪装
↓
② 服务器验证并同步伪装状态，客户端收到伪装通知
↓
③ BP_NR3E1_Changed Actor 被创建/激活，挂载 NR3E1_Changed.lua

5.2 阶段二：配置读取与材质加载

初始化流程

① 蓝图事件图表触发初始化 → 调用 GetStaticMeshComponentImp() 获取所有 StaticMesh 组件
↓
② 调用 CacheMaterialsImp() 缓存所有组件的原始材质（用于后续恢复）
↓
③ Lua 层 GetDisguiseConfig() 根据玩家 UID 和装扮数据获取伪装特效道具配置
↓
④ 判断伪装类型：IsDisguise9Using() / IsDisguise10Using()
↓
⑤ 通过 GetInVisibleCharacterMaterial() 加载第1层隐身材质（pack2.materialPath）
↓
⑥ 通过 GetInVisibleOverlayMaterial() 加载第2层叠加材质（pack3.materialPath）

5.3 阶段三：材质应用（Set Alpha 函数）

蓝图 Set Alpha 函数是材质应用的核心，包含三条并行处理分支：

分支触发条件处理逻辑
分支1：隐身状态 角色处于伪装隐身状态 ① 获取隐身材质 → ② 调用 Update Duplicate Invisible Material 设置复制模型材质 → ③ 调用 SetDisguiseOverlayMaterial() 设置 Overlay 叠加材质
分支2：普通伪装 非隐身的伪装状态 遍历 StaticMeshComponents → 获取材质数量 → 循环设置每个材质槽 → 更新复制模型材质
分支3：恢复原始 伪装解除/失效 遍历 StaticMeshComponents → 从缓存恢复原始材质 → 调用 RecoverDisguiseOverlayMaterial() 清除 Overlay

5.4 阶段四：复制模型处理（果冻特效专属）

Update Duplicate Invisible Material 函数流程

① 缓存原有材质：获取复制模型的 StaticMeshComponents，CLEAR 并重新遍历缓存
↓
② 清除旧缓存：清理之前的材质缓存数据
↓
③ 设置材质：调用 Lua 的 GetInVisibleCharacterMaterial() 获取材质 → 遍历复制模型 StaticMesh 设置隐身材质
↓
④ 设置缩放：通过 GetDeplicateModelScale() 获取 1.2 倍缩放比例，应用到 MeshRootDuplicate

5.5 阶段五：动态效果驱动

运行时动画更新

① 角色移动/跳跃时触发 MovementStateChangedDelegate
↓
② 蓝图调用 JumpStateHandle 处理跳跃状态
↓
③ Lua UpdateAnimParams(speed, moveState) 更新动画参数
↓
④ AnimInstance 根据 SpeedXY 和 MoveState 驱动果冻形变/晃动动画

5.6 阶段六：伪装解除与清理

① 触发 TryDestroyImp() 或 Start Disappear 事件
↓
② 播放消失粒子特效 FX_NR3E1_Prop_Dis... 和碎裂材质 MI_NR3E_EF_Broke...
↓
③ RecoverDisguiseOverlayMaterial() 清除 Overlay 材质
↓
④ 从缓存恢复所有 StaticMeshComponent 的原始材质
↓
⑤ Start Scale to Zero 缩放归零动画 → Actor 销毁

6. 道具配置数据结构

根据代码中的字段访问链，推断道具配置结构如下：

ItemConfig = {
    id = 241210,                        -- 果冻特效道具 ID
    nr3eDecorate = {
        pack2 = {
            materialPath = "...",       -- 主体隐身材质路径（第1层）
            model = {
                scale = 120             -- 复制模型缩放值（120 / 100 = 1.2倍）
            }
        },
        pack3 = {
            materialPath = "..."        -- Overlay叠加材质路径（第2层，透明度更低）
        }
        -- 其他可配置项（推测）：
        -- trailEffect    拖尾特效路径
        -- invisibleEffect 隐身特效路径
        -- offset          偏移参数
        -- soundEffect     伪装音效
    }
}
⚠ 配置注意：
  • pack2.model.scale 的值是百分比形式，实际缩放需除以 100
  • 材质路径必须是有效的 UE4 资源路径，否则 AssetLoadUtils.LoadAssetObject() 返回 nil
  • 空字符串 "" 的材质路径会被 Lua 层过滤，不会执行加载

10. 已知问题与注意事项

⚠ 问题1：IsDisguise9Using 与 IsDisguise10Using 实现相同
两个方法均判断 ItemConfig.id == 241210，"复制模型"判定应该对应不同的道具 ID。当前可能是临时占位或 BUG，需确认是否需要区分两种伪装类型。
⚠ 问题2：材质加载无错误处理
