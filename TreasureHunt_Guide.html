📋 一、新手引导系统概述

寻宝模式的新手引导基于通用的 PlayerGuideModel 实现，核心流程如下：

核心逻辑：
1. 通过 HasFinishGuide(guideID) 判断玩家是否已完成某个引导步骤
2. 如果未完成，则通过 DispatchEvent 派发引导事件名称
3. 事件名称对应新手引导配置表中的 "触发事件"，引导系统监听该事件后展示引导UI
4. 引导分为 平民(Prey) / 狼人(Hunter) 两个阵营独立配置

PlayerGuideModel 关键方法

方法说明
HasFinishGuide(guideID)先检查AB测试分组，再检查本地缓存和服务端已完成列表
IsGuideAvailable(guideID)判断引导是否可用（未完成且满足前置条件）
StartGuide(guideID)开始某个引导
EndCurrentGuide(withComplete)结束当前引导，withComplete=true 表示标记完成
GetCurrentGuideID()获取当前正在进行的引导ID

定义文件：LetsGo/Content/LetsGo/Script/Model/Guide/PlayerGuideModel.lua

⚙️ 四、引导触发时机与代码位置详解

📌 阶段1：游戏开局引导

触发时机：游戏开始时的 "开始界面"（StartView）关闭后
触发条件：HasFinishGuide(503052/503060) == false
引导ID：503052 平民 503060 狼人

代码文件：UI_InLevel_NR3E3_TreasureHunt_StartView.lua → OnClose()

function UI_InLevel_NR3E3_TreasureHunt_StartView:OnClose()
    -- ... 清理定时器 ...
    if not _MOE.Models.PlayerGuideModel:HasFinishGuide(503052) then
        _MOE.EventManager:DispatchEvent("ON_NR3E3TreasureHuntRule_Prey_StartGameGuide")
    end
    if not _MOE.Models.PlayerGuideModel:HasFinishGuide(503060) then
        _MOE.EventManager:DispatchEvent("ON_NR3E3TreasureHuntRule_Hunter_StartGameGuide")
    end
end
StartView 是游戏开局时显示的身份确认界面（显示10秒后自动关闭或玩家手动关闭），界面关闭时触发开局引导。

📌 阶段2：挖宝引导

触发时机：玩家靠近宝箱，挖宝技能按钮出现在UI上时
触发条件：HasFinishGuide(503053/503061/503070/503072) == false
结束时机：挖宝技能按钮从UI上隐藏时，或PC端按下快捷键成功挖宝时
引导ID：503053/503070 503061/503072

代码文件：VocationAbility_CTH_OpenTreasureBox.lua

function VocationAbility_CTH_OpenTreasureBox:OnVocationDataHighLightOnButton()
    if _MOE.GEnv.IsPCStyle then
        -- PC端引导
        if not HasFinishGuide(503070) then DispatchEvent("Prey_DiggingTreasureGuide_PC") end
        if not HasFinishGuide(503072) then DispatchEvent("Hunter_DiggingTreasureGuide_PC") end
    else
        -- 移动端引导
        if not HasFinishGuide(503053) then DispatchEvent("Prey_DiggingTreasureGuide") end
        if not HasFinishGuide(503061) then DispatchEvent("Hunter_DiggingTreasureGuide") end
    end
end
⚡ PC端特殊处理：在 K2_ActivateAbility 中，PC端通过 GetCurrentGuideID() 检测当前引导，如果匹配则直接调用 EndCurrentGuide(true) 完成引导。

📌 阶段7：查看宝物提示引导

触发时机：商店购买物品后，飞行特效（Trail）动画播放结束关闭时
触发条件：HasFinishGuide(503059/503067) == false
引导ID：503059 503067

代码文件：UI_InLevel_NR3E3_TreasureHunt_Trail.lua → OnClose()

function UI_InLevel_NR3E3_TreasureHunt_Trail:OnClose()
    self.Data = nil
    _NR3E.TimerManager:ClearRefTimers(self)
    if not _MOE.Models.PlayerGuideModel:HasFinishGuide(503059) then
        _MOE.EventManager:DispatchEvent("ON_NR3E3TreasureHuntRule_Prey_CheckTreasureGuide")
    end
    if not _MOE.Models.PlayerGuideModel:HasFinishGuide(503067) then
        _MOE.EventManager:DispatchEvent("ON_NR3E3TreasureHuntRule_Hunter_CheckTreasureGuide")
    end
end
Trail 是商店购买物品后的飞行特效UI（贝塞尔曲线飞行动画），飞行特效结束后触发引导。

💡 五、教学提示系统（TeachingTips）

除了新手引导配置表驱动的引导外，寻宝模式还有一套 "教学提示"（TeachingTips） 系统，基于 NR3E3TipsModel 实现。

TeachingTipsType 枚举定义

定义在 NR3E3GameConfig.lua 第1314行起：

枚举值名称说明触发位置
1000TreasureProgressReach达到寻宝进度UI_InLevel_NR3E3_TreasureHunt_Task.lua → UpdateUI()
1001TreasureCarryDelay搬运宝物持续3秒NR3E3_VocationAbility_CTH_TreasureDrop_New.lua 第56行
1002TreasureMeetingEndToMain会议结束回到主界面由游戏流程自动触发
1003TreasureAttack摧毁宝物NR3E3AbilityActor_TreasureBase.lua 第66行

🎯 TreasureProgressReach (1000)

触发时机：寻宝进度条动画播放完成后（alpha >= 1.0）

检查逻辑：CheckTaskProgressReach（复用普通任务进度检查）

📦 TreasureCarryDelay (1001)

触发时机：玩家搬运宝物持续3秒后

检查逻辑：CheckDefault（默认通过）

🏛️ TreasureMeetingEndToMain (1002)

触发时机：由游戏流程自动触发

检查逻辑：CheckTreasureMeetingEndToMain

  • conditionParams[1] == 0 → 未搬运宝物时触发
  • conditionParams[1] == 1 → 正在搬运宝物时触发

⚔️ TreasureAttack (1003)

触发时机：服务端→客户端 通知宝物被摧毁时

检查逻辑：CheckDefault（默认通过）
